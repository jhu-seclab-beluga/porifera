# Instrumentation Module – Design Specification

## Design Overview

The Instrumentation module modifies PHP source code by injecting probe function wrappers around target expressions and provides restoration. The wrapping strategy is injectable: `StandardProbeStrategy` skips unsafe contexts; `ElevatingProbeStrategy` wraps a safe ancestor expression instead.

Probe function names are auto-generated by `InstrumentationManager` with prefix `__porifera_probe_` (e.g. `__porifera_probe_a1b2c3d4`). The name is managed solely by the Manager and passed to `ASTInstrumenter`; strategies never see it. Deinstrumentation matches by prefix.

- **Classes**: `InstrumentationManager` (internal), `InstrumentationRegistry` (internal), `ASTInstrumenter` (internal), `ASTDeinstrumenter` (internal), `ProbeStrategy` (internal, abstract), `StandardProbeStrategy` (internal), `ElevatingProbeStrategy` (internal)
- **Relationships**: `InstrumentationManager` contains `InstrumentationRegistry`, `InstrumentationManager` uses `ASTInstrumenter`, `InstrumentationManager` uses `ASTDeinstrumenter`, `ASTInstrumenter` uses one `ProbeStrategy` (injected)
- **Abstract**: `ProbeStrategy` (implemented by `StandardProbeStrategy`, `ElevatingProbeStrategy`)
- **Exceptions**: `InstrumentationError`, `DeinstrumentationError` extends `InstrumentationError`
- **Dependency roles**: Orchestrator: `InstrumentationManager`. Helpers: `ASTInstrumenter`, `ASTDeinstrumenter`, `InstrumentationRegistry` (inputs by argument). Strategies: `ProbeStrategy` (injectable into `ASTInstrumenter`).
- **Shared Components**: `php_parser_py.AST`
- **Resources**: `resources/porifera_runtime.php`

## Class / Type Specifications

### InstrumentationRegistry Class (Internal)
- **Responsibility**: Persists instrumentation state to `.porifera_registry.json` for precise restoration during deinstrumentation.
- **State**:
  - `registry_path: Path` — Path to `.porifera_registry.json`
  - `data: Dict[str, List[str]]` — Maps file paths to lists of `expr_key` strings
- **[__init__(registry_path: Path)]**
  - **Behavior**: Sets path, loads existing data from disk if present.
  - **Input**: `registry_path`
  - **Output**: None
- **[register(file_path: Path, expr_key: str) -> None]**
  - **Behavior**: Appends `expr_key` to the file's list, persists to disk atomically.
  - **Input**: `file_path`, `expr_key`
  - **Output**: None
  - **Raises**: `OSError` on write failure
- **[get_expr_keys(file_path: Path) -> List[str]]**
  - **Behavior**: Returns expr_keys for a file from in-memory data.
  - **Input**: `file_path`
  - **Output**: List of expr_key strings (empty if none)
- **[get_all_files() -> List[Path]]**
  - **Behavior**: Returns all instrumented file paths.
  - **Input**: None
  - **Output**: List of paths
- **[clear() -> None]**
  - **Behavior**: Deletes registry file and clears in-memory data.
  - **Input**: None
  - **Output**: None

**Example Usage**:
```python
registry = InstrumentationRegistry(project_root / ".porifera_registry.json")
registry.register(file_path, "total")
expr_keys = registry.get_expr_keys(file_path)
```

### ProbeStrategy Class (Internal, Abstract)
- **Responsibility**: Selects which node to wrap. Pure decision logic — no AST mutation, no probe naming.
- **[select_wrap_target(ast: AST, node: Node, wrapped_node_ids: set[str]) -> Optional[Node]]** (abstract)
  - **Behavior**: Given a target node, returns the node that should be wrapped, or None if unable to instrument. The returned node may differ from the input (e.g. an ancestor for elevation).
  - **Input**: `ast`, `node` (target), `wrapped_node_ids` (IDs already wrapped — skip these)
  - **Output**: Node to wrap, or None
  - **Raises**: None

### StandardProbeStrategy Class (Internal)
- **Responsibility**: Selects safe rvalue targets directly; returns None for unsafe targets.
- **[select_wrap_target(ast, node, wrapped_node_ids) -> Optional[Node]]**
  - **Behavior**: If `_is_safe_to_wrap(ast, node)` is True and `node.id not in wrapped_node_ids` → return `node`. Otherwise → return None.

### ElevatingProbeStrategy Class (Internal)
- **Responsibility**: Selects safe targets directly; for unsafe targets, walks up to the nearest wrappable `Expr_*` ancestor.
- **[select_wrap_target(ast, node, wrapped_node_ids) -> Optional[Node]]**
  - **Behavior**: If safe and `node.id not in wrapped_node_ids` → return `node`. If unsafe → walk parent chain until hitting a non-`Expr_*` node (statement boundary), find first ancestor where `_is_safe_to_wrap()` is True and `ancestor.id not in wrapped_node_ids` → return `ancestor`. No ancestor found → return None.

**Example Usage**:
```python
standard = StandardProbeStrategy()
elevating = ElevatingProbeStrategy()
```

### ASTInstrumenter Class (Internal)
- **Responsibility**: Owns probe function name and wrapping logic (`_wrap_node`). Uses `ProbeStrategy` to select wrap targets; tracks wrapped nodes to prevent double-wrapping.
- **State**:
  - `project_ast: AST` — Parsed project AST
  - `_strategy: ProbeStrategy` — Injected target selection strategy
  - `_probe_func_name: str` — Auto-generated probe function name (from Manager)
  - `_wrapped_nodes: set[str]` — IDs of already-wrapped nodes
- **[__init__(project_ast: AST, strategy: ProbeStrategy, probe_func_name: str)]**
  - **Behavior**: Stores AST, strategy, probe name; initializes empty `_wrapped_nodes`.
  - **Input**: `project_ast`, `strategy`, `probe_func_name`
  - **Output**: None
- **[instrument_node(node_id: str, expr_key: str) -> bool]**
  - **Behavior**: Looks up node by ID. Calls `_strategy.select_wrap_target()` to get the node to wrap. If None, logs warning and returns False. Otherwise, calls `_wrap_node()` on the selected target, adds its ID to `_wrapped_nodes`, returns True.
  - **Input**: `node_id`, `expr_key`
  - **Output**: True if instrumented, False otherwise
  - **Raises**: `InstrumentationError` on AST errors; `KeyError` if node ID not found
- **[_wrap_node(ast: AST, node: Node, expr_key: str) -> None]** (private)
  - **Behavior**: Creates probe `Expr_FuncCall` AST structure using `self._probe_func_name`, re-parents target node under the call. Line properties cloned from target node.
  - **Input**: `ast`, `node`, `expr_key`
  - **Output**: None
  - **Raises**: `InstrumentationError` if node has no parent edge

### ASTDeinstrumenter Class (Internal)
- **Responsibility**: Removes probe function calls from AST by unwrapping to restore original expressions. Identifies probes by `__porifera_probe_` prefix matching.
- **[unwrap_probe_ast(file_path: Path, expr_key: str, ast, exclude_node_ids: Optional[set] = None) -> bool]**
  - **Behavior**: Finds probe call matching `expr_key` in first argument, replaces with second argument (original expression). Returns True if found and replaced.
  - **Input**: `file_path` (for error messages), `expr_key`, `ast` (modified in place), `exclude_node_ids`
  - **Output**: True if unwrapped, False if not found
  - **Raises**: `DeinstrumentationError` if structure invalid
- **[scan_and_unwrap(project_root: Path) -> List[Path]]**
  - **Behavior**: Scans all PHP files, fast-path filters by `__porifera_probe_` text search, parses and unwraps all probe calls.
  - **Input**: `project_root`
  - **Output**: List of modified file paths
  - **Raises**: `DeinstrumentationError` on errors

### InstrumentationManager Class (Internal)
- **Responsibility**: Orchestrates instrumentation lifecycle: resolves file membership from AST, delegates wrapping to `ASTInstrumenter`, manages file regeneration, runtime helper injection, and registry.
- **State**:
  - `project_ast: AST` — Parsed project AST
  - `registry: InstrumentationRegistry` — State tracker
  - `_instrumenter: ASTInstrumenter` — AST instrumentation handler
  - `_deinstrumenter: ASTDeinstrumenter` — AST deinstrumentation handler
- **[__init__(project_ast: AST, strategy: Optional[ProbeStrategy] = None, output_dir: Optional[Path] = None)]**
  - **Behavior**: Derives project root from AST. Creates `InstrumentationRegistry`. Auto-generates probe function name (`__porifera_probe_{hex}`) and a unique timestamp string (`YYYYMMDD_HHMMSS_{hex6}`). If `strategy` is None, creates `StandardProbeStrategy()` as default. Creates `ASTInstrumenter(project_ast, strategy, probe_func_name)` and `ASTDeinstrumenter()`. Stores `output_dir` for runtime helper generation.
  - **Input**: `project_ast`, `strategy` (optional), `output_dir` (optional — directory for probe output; defaults to project root via PHP `__DIR__`)
  - **Output**: None
- **[instrument(targets: Dict[str, str]) -> List[Path]]**
  - **Behavior**: Returns `[]` if targets empty. Copies runtime helper to project root if not present. Resolves file membership for each target node via AST ancestor traversal. Groups targets by file. For each file: calls `_instrumenter.instrument_node()` per target, registers expr_keys in registry, regenerates file via `PrettyPrinter`, injects `require_once` for runtime helper. Returns modified file paths.
  - **Input**: `targets` — maps `node_id → expr_key`
  - **Output**: List of modified file paths
  - **Raises**: `InstrumentationError` on failure
- **[deinstrument(use_registry: bool = True) -> List[Path]]**
  - **Behavior**: If `use_registry` is True, uses registry for precise restoration. If False, scans all PHP files (fail-safe). Removes runtime helper and clears registry.
  - **Input**: `use_registry`
  - **Output**: List of restored file paths
  - **Raises**: `DeinstrumentationError` on failure

**Example Usage**:
```python
from porifera._manager import InstrumentationManager
from porifera._ast_operations import ElevatingProbeStrategy

project_ast = parse_file("project.php")

# Default: StandardProbeStrategy
manager = InstrumentationManager(project_ast)
modified = manager.instrument({"node_123": "total", "node_456": "count"})
restored = manager.deinstrument()

# With elevation
manager = InstrumentationManager(project_ast, strategy=ElevatingProbeStrategy())
```

## Function Specifications

### Module: `porifera`

**[instrument(targets: Dict[str, str], project_ast: AST, strategy: Optional[ProbeStrategy] = None, output_dir: Optional[Path] = None) -> List[Path]]**
- **Responsibility**: Public API for applying runtime probes to target expressions.
- **Behavior**: Creates `InstrumentationManager` internally, delegates to `manager.instrument(targets)`.
- **Input**:
  - `targets`: Maps `node_id → expr_key`
  - `project_ast`: Parsed project AST
  - `strategy`: Wrapping strategy (default: `StandardProbeStrategy`)
  - `output_dir`: Directory for probe output file (default: project root via PHP `__DIR__`)
- **Output**: List of modified file paths
- **Raises**: `InstrumentationError`

**[deinstrument(project_ast: AST, use_registry: bool = True) -> List[Path]]**
- **Responsibility**: Public API for removing runtime probes.
- **Behavior**: Creates `InstrumentationManager` internally, delegates to `manager.deinstrument(use_registry)`.
- **Input**:
  - `project_ast`: Parsed project AST
  - `use_registry`: Registry mode (True) or scan mode (False)
- **Output**: List of restored file paths
- **Raises**: `DeinstrumentationError`

**Example Usage**:
```python
from porifera import instrument, deinstrument
from porifera._ast_operations import ElevatingProbeStrategy

project_ast = parse_file("project.php")

modified = instrument({"node_123": "total"}, project_ast)
restored = deinstrument(project_ast)

# With elevation
modified = instrument({"node_123": "total"}, project_ast, strategy=ElevatingProbeStrategy())
```

### Module: `porifera._ast_operations`

**[_is_safe_to_wrap(ast: AST, node: Node) -> bool]** (module-level, private)
- **Responsibility**: Checks if wrapping a node in a function call preserves semantics.
- **Behavior**: Finds incoming `PARENT_OF` edge, extracts parent node type and edge field, checks `(parent_type, edge_field)` against `_UNSAFE_WRAP_CONTEXTS`.
- **Input**: `ast`, `node`
- **Output**: True if safe, False if unsafe or no parent edge
- **Raises**: None

## Exception / Error Types

**InstrumentationError**: Raised during instrumentation (parse error, write failure, registry error, runtime helper not found).

**DeinstrumentationError**: Extends `InstrumentationError`. Raised during deinstrumentation (registry corrupted, file modified externally, AST parse error).

---

## Validation Rules

### Wrap Safety
- `_is_safe_to_wrap(ast, node)` is a module-level function checking `(parent_node_type, edge_field)` against `_UNSAFE_WRAP_CONTEXTS`. Used by both strategies.
- **Unsafe lvalue contexts** (Category A): `Expr_Assign.var`, `Expr_AssignRef.var`, all `Expr_AssignOp_*.var`, `Expr_PreInc.var`, `Expr_PostInc.var`, `Expr_PreDec.var`, `Expr_PostDec.var`, `Stmt_Foreach.keyVar`, `Stmt_Foreach.valueVar`, `Stmt_Unset.vars`.
- **Unsafe reference contexts** (Category C): `Expr_AssignRef.expr` — must be a variable, not a function call.
- **Safe for-loop contexts**: `Stmt_For.init` and `Stmt_For.loop` are NOT in the unsafe set — wrapping expressions preserves side effects; for-loop discards return values.
- **Strategy behavior**: `StandardProbeStrategy` skips unsafe targets with `logger.warning`. `ElevatingProbeStrategy` walks up to find a wrappable `Expr_*` ancestor.
- **Un-instrumentable**: `Stmt_Foreach.keyVar/valueVar`, `Stmt_Unset.vars` — statement-level positions with no wrappable expression ancestor. Skipped by both strategies.
- **Double-wrap prevention**: `ASTInstrumenter` tracks wrapped node IDs in `_wrapped_nodes` set; passed to strategy for checking.
- **Known limitation**: By-ref function arguments require callee signature resolution; not detectable at AST layer.
- **Known limitation**: `Expr_List` destructuring targets require multi-level ancestor check; relies on upstream filtering.

### Probe Identification
- Probe function names use prefix `__porifera_probe_` followed by 8-char hex (e.g. `__porifera_probe_a1b2c3d4`).
- Deinstrumentation identifies probes by prefix matching — no need to persist the exact function name.
- Fast-path text search in scan mode uses `__porifera_probe_` as filter string.

### Safety
- All modified files must pass `php-parser-py` parse checks before writing.
- Registry file updates are atomic (write-then-rename).
- Scan mode (`use_registry=False`) is fail-safe fallback when registry is missing or corrupted.

### Runtime Helper
- Fixed filename: `porifera_runtime.php`. Copied to project root only if not already present.
- `require_once` statement injected into instrumented files.
- Probe output file: `.porifera_data_{timestamp}.jsonl`. Each instrumentation run generates a unique timestamp (`YYYYMMDD_HHMMSS_{hex6}`) to avoid overwriting previous data.
- Output directory is configurable via `output_dir` parameter (defaults to PHP `__DIR__`).
- PHP-side concurrency safety: uses `fopen` + `flock(LOCK_EX)` + `fwrite` instead of `file_put_contents`.

### Deinstrumentation Modes
- **Registry mode** (`use_registry=True`): Precise restoration using registry data. Faster.
- **Scan mode** (`use_registry=False`): Scans all PHP files. Works without registry. Slower but robust.
